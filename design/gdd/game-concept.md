## Elevator Pitch

> 一款怀旧像素风的回合制爬塔游戏:你在一座塔里开门、捡钥匙、打怪,用确定性的战斗一层层往上爬,亲眼看着自己的数值滚雪球式变强。
>
> 它是抖音小游戏的"经典魔塔",砍掉烧脑负担、放大变强爽感,即点即玩。

## Core Fantasy

我在一层层征服这座塔,而且肉眼可见地越来越强。

每开一扇门、每打一个怪,玩家都立刻得到"变强"的实感:伤害飘字跳出、属性数字上涨、升级闪光。核心情感承诺不是"解开难题"(那是硬核魔塔),而是**掌控 + 成长的滚雪球爽感**——配上怀旧像素皮,唤起对红白机时代魔塔的记忆。

---

## Unique Hook

像经典《魔塔》一样开门、用钥匙、看伤害预演打怪,**而且**每一步都有立竿见影的数值爆发反馈,把"算路线的解谜游戏"重塑为"看着自己变强的爽游戏"。

差异点在于**设计取向**:保留魔塔确定性战斗的"可预演、可掌控",但把平衡曲线和反馈表现调向"成长爽快"而非"最优解烧脑"。这是机制层面的取舍,不是换皮。

## Player Experience Analysis (MDA Framework)

### Target Aesthetics (What the player FEELS)

| Aesthetic | Priority | How We Deliver It |
| ---- | ---- | ---- |
| **Sensation** (sensory pleasure) | 1 | 伤害飘字、升级闪光、属性跳涨、像素打击反馈 |
| **Challenge** (obstacle course, mastery) | 2 | 半策略确定性战斗:选打谁/先开哪扇门 |
| **Discovery** (exploration, secrets) | 3 | 开门揭图、宝箱、未知楼层 |
| **Submission** (relaxation, comfort zone) | 4 | 低门槛、即点即玩、节奏明快 |
| **Fantasy** (make-believe) | 5 | 勇者爬塔的经典幻想 |
| **Narrative** (drama, story arc) | N/A | 极简,几乎无叙事 |
| **Fellowship** (social connection) | N/A | 单机,最多排行榜 |
| **Expression** (self-expression) | N/A | 不做 build 自定义 |

### Key Dynamics (Emergent player behaviors)
- 玩家会在开门前权衡顺序(先拿钥匙还是先打怪),形成轻度路线推演。
- 玩家追逐"再上一层就更强一截"的即时反馈,产生"再爬一层"心理。
- 玩家自愿看激励广告换碎片翻倍 / 卡关复活补偿,而非被打断;碎片积累带来"再清一层多攒一点"的额外驱动。

### Core Mechanics (Systems we build)

1. **网格移动与开门** —— 棋盘式格子移动,钥匙(黄/蓝/红)开对应门。
2. **确定性回合战斗** —— 无随机数,撞怪即逐回合结算,战前显示伤害预演;玩家通过选择先后顺序进行轻策略。
3. **数值成长系统** —— HP/攻/防 通过药水、打怪、升级提升;滚雪球曲线。
4. **数值反馈表现层(juice)** —— 伤害飘字、升级闪光、属性跳涨,这是"手感(feel)"的核心。
5. **钥匙碎片软通货(经济层)** —— 打怪/探索掉碎片,碎片可在层间换属性/药水/补钥匙;是滚雪球在 HP/攻/防 之外的**第二个出口**,也是激励广告的经济载体。详见 Monetization Model 节。
6. **容错机制(防死局)** —— 每层保底资源 + 卡关后"碎片换属性"的出路,保证轻用户不撞死墙(见 Pillar 2)。
7. **激励式广告变现** —— 碎片翻倍 + 复活(转一次性属性补偿,让你真能过那个怪),玩家自愿触发。

## Player Motivation Profile

### Primary Psychological Needs Served

| Need | How This Game Satisfies It | Strength |
| ---- | ---- | ---- |
| **Autonomy** | 选择开门顺序、战斗路线、资源分配 | Supporting |
| **Competence** | 肉眼可见的数值增长与变强反馈 | Core |
| **Relatedness** | 弱;靠排行榜补一点社交连接 | Minimal |

### Player Type Appeal (Bartle Taxonomy)

- [x] **Achievers** —— 收集钥匙/道具、推进楼层、数值进度,主要受众。
- [x] **Explorers** —— 开门揭图、找宝箱、探索未知楼层,次要受众。
- [ ] **Socializers** —— 不服务(单机)。
- [ ] **Killers/Competitors** —— 弱;仅排行榜略沾边。

### Flow State Design

- **Onboarding curve**:零教程可懂,前 3 层即教学(撞怪=战斗、捡钥匙=开门)。
- **Difficulty scaling**:楼层越高怪越强,但成长曲线 + 容错机制保证"算得过来就打得过",轻用户不撞死墙。
- **Feedback clarity**:每次行动都有数字与画面反馈,玩家始终知道自己在变强。
- **Recovery from failure**:卡关有容错出路(保底资源 / 碎片换属性),或自愿看广告换复活补偿;失败不是死局,是"差一点,再攒一攒就过"。

## Core Loop

### Moment-to-Moment (30 seconds)
看一眼当前层 → 决定开哪扇门 / 打哪个怪(可见伤害预演)→ 撞击结算 → 伤害飘字 + 升级闪光 + 属性跳涨 → 推进一格。

### Short-Term (5-15 minutes)
清完一层 → 走楼梯上行 → 新层有更强的怪与更香的宝箱 → "再上一层变强一截"的钩子驱动持续推进。

### Session-Level (30-120 minutes)
从底层爬到 Boss 层(或无尽塔的一个段落)。自然存档点在每层楼梯。卡关或死亡时可看广告复活/拿增益。一段游玩 = 推进若干层 + 一次明显的实力跃迁。

### Long-Term Progression
属性(HP/攻/防)+ 钥匙/道具滚雪球。无尽塔模式提供"爬得更高 / 刷新纪录"的长期目标;排行榜提供社交对比。

### Retention Hooks
- **Curiosity**:上面的楼层有什么怪、什么宝?未揭开的塔顶。
- **Investment**:已爬到的高度、积累的数值,不想丢失。
- **Social**:排行榜排名。
- **Mastery**:更优的开门/战斗顺序,刷新爬塔纪录。

## Game Pillars

> **真北(North Star)= Pillar 1「看得见的成长」。** 受众押抖音轻用户——他们要的是"爽",
> 不是"算"。确定性(Pillar 2)从卖点**降级为支撑手段**:它服务于"成长可预期、可掌控",
> 而非要求玩家解最优题。当 Pillar 1 与 Pillar 2 冲突时,**优先 Pillar 1**。

### Pillar 1: 看得见的成长 ⭐ (真北)
每一次战斗、每一层推进,玩家都能立刻在数字与画面上看到自己变强。

*Design test*:任何抉择,选更能让玩家**直观感到变强**的那个。这是裁决其他支柱冲突的最高准则。

### Pillar 2: 确定性 + 容错(支撑 Pillar 1)
战斗无随机数、可预演,让成长可掌控、可预期。**但纯确定性会让轻用户撞死墙**,所以配一层容错:
每层保底资源、卡关后有"碎片换属性"的出路——确定性是为了"算得过来就打得过",不是为了卡死玩家。

*Design test*:加随机会破坏"战斗可预演"吗?会就别加(随机只允许在地图/事件层,见下)。但若"严格确定"
导致轻用户死局,优先加容错而非维持纯确定。

### Pillar 3: 三秒上手
抖音用户即点即玩,核心操作一根手指、零教程可懂。

*Design test*:纠结要不要加新系统/按钮时,选认知负担更小的那个。

### Pillar 4: 每层都有新发现
每爬一层都给玩家一个新的"哦?"——新怪、新道具、新地形机关、新事件——维持探索好奇与重玩动力。

*Design test*:一个新楼层若只是"同样的怪数值更高",它没达标;每层至少要有一个新的可发现元素。
(与 Pillar 3「三秒上手」张力:新发现 vs 认知简洁——用"视觉自解释"的新元素化解。)

### Anti-Pillars (What This Game Is NOT)

- **NOT 复杂多角色/装备搭配系统**:会破坏"三秒上手",且首作几周做不完。
- **NOT 随机数战斗(暴击/闪避概率)**:会破坏"战斗可预演"。随机**只允许在地图/事件层**
  (随机楼层布局、事件房、碎片掉落量),战斗结算本身保持确定。
- **NOT 强制/打断式广告**:只做激励式(碎片翻倍 / 复活补偿,玩家自愿),保护"明快推进"的爽感。
- **NOT 加速类广告**:回合制无被动流逝时间可加速,跳过战斗动画反而破坏核心 feel。
- **NOT 联网/社交/PVP**:首作几周做不完,且偏离单机爬塔核心。

## Monetization Model (变现模型)

> 由 design-review 新增。核心结论:**确定性单机若无可累积资源,激励广告无经济载体**("翻倍翻什么、加速加什么"都悬空,"复活"在确定性里也无效)。解法是引入一层最轻的软通货。

**钥匙碎片(软通货)**
- **Faucet(产出)**:打怪、探索宝箱、清层奖励掉落碎片(确定性数量;随机仅可影响"事件房额外掉落",不影响保底)。
- **Sink(消耗)**:层间用碎片换药水/临时属性/补一把钥匙;作为滚雪球在 HP/攻/防 之外的**第二个成长出口**。
- 与支柱:碎片是"看得见的成长"的延伸(Pillar 1);克制设计,不做复杂商店(Pillar 3)。

**激励广告位(玩家自愿,绝不打断)**
1. **碎片翻倍** —— 清层/拾取结算时可选看广告让本次碎片 ×2(主力变现位,有明确经济载体)。
2. **复活转属性补偿** —— 死亡时看广告,**不是原地满血复活**(确定性下复活还是打不过),而是获得一次性属性补偿,让玩家真能跨过卡住的那个怪。

**明确放弃**:加速类广告(回合制无被动时间可加速,跳动画破坏 feel)。

**排期**:碎片软通货 + 广告位在 Vertical Slice 引入;**作为 /map-systems 的独立子系统设计,不可后期外挂**。MVP 不含变现。

## Risks and Open Questions

> 本节经 design-review(5 专家)更新。根因被识别为两条:**(A)"确定性+滚雪球+无随机经济"互相绞杀**
> (碾压或死局,且砍掉变现载体);**(B)原型 1 人自测把"真"假设当"已证"**。下方标注各风险的缓解状态。

### Design Risks
- **[根因 A · 缓解中]** 确定性 + 滚雪球数学上易双极化(碾压 / 死局)。缓解:① 加容错机制(保底+碎片换属性出路);
  ② 引入碎片软通货分散滚雪球出口;③ 第二轮原型用真用户调曲线,而非文档里前置定死数值。
- 反馈表现层(juice)做不到位,"数值爆发爽感"立不住——原型已初步验证 juice 有效(PRELIMINARY)。
- 长期机械化 / 重玩性低:确定性 + 手工关卡 + 有限怪物 → 重玩钩子弱。靠 Pillar 4「每层有新发现」+ 后期随机事件层缓解。

### Technical Risks
- Godot 抖音适配器对 4.6.3 的支持需实测;若仅支持 ~4.5,可能需降版本。**行动:/setup-engine 阶段做裸导出 spike。**
- WASM 体积可能超 50MB;裁剪模块需 custom build(对首作非平凡)。
- 抖音个人开发者资质 / 广告 SDK 接入门槛待核实。**行动:现在就去抖音开发者平台注册并提交资质申请(审核是日历时间)。**
- **[根因 A 衍生]** 随机楼层(Alpha+)需可解性验证器,实现成本未计入当前 solo 估算。MVP/VS 用手工关卡规避。

### Market Risks
- 抖音魔塔/放置品类竞争激烈,自然流量难,可能依赖买量。
- **[已决策]** 受众裂缝(怀旧 mid-core vs 抖音轻用户)——已选择**押抖音轻用户为真北**,确定性降为支撑,消除"两边都不讨好"风险。
- **[根因 A · 已缓解]** 确定性单机无软资源 → 广告变现天花板极低。缓解:引入碎片软通货作为广告经济载体。

### Scope Risks
- 首作几周:像素素材产能与关卡(确定性平衡)设计可能成为瓶颈。
- 隐藏工作量被低估(QA 警示):导出测试、低端机真机测试、广告 SDK 接入。MVP 估算需加 QA buffer。

### Open Questions
- 确定性 + 容错下,真实抖音轻用户会不会撞墙?→ **第二轮原型(N≥3 真人,加长到 8–10 层 + 难度拐点)验证**(Vertical Slice 前)。
- 抖音 Godot 适配器对 4.6.3 的实际支持 + WASM 体积?→ /setup-engine 阶段导出 spike。
- 个人开发者能否上架并接广告?→ 抖音开发者平台核实资质(现在就申请)。
- 碎片软通货的具体 faucet/sink 数值?→ /map-systems + /design-system 阶段设计。

## MVP Definition

**Core hypothesis(可测版)**:在 N≥3 名**目标用户**(18–40 岁手机休闲玩家,**非开发者**)的受控游玩测试中,确定性爬塔 + 数值爆发反馈的循环能驱动主动推进与"变强"感知。**测量信号与阈值**:
- **信号 1(主动推进)**:≥80% 测试者在完成第 1 层后,未经引导主动进入第 2 层。
- **信号 2(会话延长)**:≥60% 测试者被告知"可以结束了"时,表示想继续/再爬一层。
- **信号 3(变强感知)**:≥80% 测试者在开放访谈("玩完什么感受?")中自发提到数值成长/越打越顺。
- **失败条件(任一触发即 FAILED)**:信号1<60% / 信号2<40% / 信号3<60% / >2 名测试者在第 2 层前主动关掉。

**MVP 成功标准(进入 Vertical Slice 前须全满足)**:
- *功能完整性(可自测)*:3 层无 crash/卡死;战斗结算与伤害预演 100% 一致(无 RNG);HP 归零正确触发死亡;升级/拾取后属性在 UI 可见跳涨。
- *用户体验信号(需 ≥3 名非开发者)*:见上方 3 个信号阈值;0 名测试者"不知道在干嘛"超过 30 秒。
- *数值健康性(可自动化测试)*:战斗公式 `max(1,ATK−DEF)` 对所有怪物结算 ≥1;推荐路线通关 3 层后 HP>0;无钥匙死锁。

**Required for MVP**:
1. 3–5 层**手工固定**单塔,网格移动 + 楼梯上下行(可回层补打/补捡)。
2. 钥匙(至少黄钥匙)+ 门;至少 1 套像素素材。
3. 确定性**逐回合**战斗 + 战前伤害预演(战斗动画间隔默认 0.3s/回合,来源:原型)。
4. HP/攻/防 成长 + 数值反馈表现(飘字/升级闪光/属性跳涨)。

**Explicitly NOT in MVP**:
- 激励广告、碎片软通货、排行榜、随机楼层/事件、无尽塔。
- 多钥匙颜色、多怪物种类、Boss。
- 音效/音乐打磨。

## Visual Identity Anchor

**视觉方向**:8-bit 怀旧魔塔

**一句话视觉规则**:一切都用低分辨率像素 + 有限调色板,致敬 FC/红白机时代的魔塔。

**支撑视觉原则**:
1. **网格即美术** —— 角色、怪物、道具、墙体都对齐 16×16 像素格,棋盘式清晰。*设计测试:任何新素材必须能在 16×16 网格里读得清。*
2. **有限调色板** —— 全局锁定一个复古色板(~16–32 色)。*设计测试:新增颜色前先问"能不能复用现有调色板"。*
3. **数值反馈高对比** —— 伤害飘字、升级闪光用高饱和亮色,在朴素像素底上跳出来。*设计测试:成长反馈的视觉必须比环境更亮、更跳。*

**色彩哲学**:暗色石墙塔身为底,高饱和的金(钥匙/宝)、红(怪/伤害)、蓝(魔法/护盾)作为功能性强调色——怀旧但反馈清晰。

> 本节是 Art Bible 的种子,捕捉"怀旧像素 + 高对比反馈"这一核心视觉取舍,供 /art-bible 展开。
