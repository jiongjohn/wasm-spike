## 1. 层级承诺

本项目以 **WCAG-AA 为 BLOCKING 基线**：下列「承诺（Commit）」项是 Pre-Production 及之后所有 UI/UX story 的硬门槛，未满足不可标 Done。少数项列为「期望（Aspire）」——非 MVP 阻断，VS/Alpha 评估。

| 维度 | Commit（BLOCKING） | Aspire（非阻断） |
|------|--------------------|-------------------|
| 视觉 | 对比度 ≥4.5:1；色盲双信道；字 ≥12pt | 可调字号、对比度 ≥7:1（AAA） |
| 运动 | 触控热区 ≥44×44px；无时间压迫手势；输入宽容 | 触控热区可放大选项 |
| 认知 | 三秒上手；渐进披露；无阅读限速 | 多语言朗读辅助 |
| 听觉 | 无唯音频承载的关键信息 | 全面字幕系统 |
| 动效 | 关键反馈不依赖快速闪烁；闪烁 ≤3Hz | 完整 reduced-motion 全局开关 |
| 辅助技术 | —（抖音小游戏 WebView 无标准屏幕朗读支持） | AccessKit 屏幕朗读（4.5+，Aspire） |

## 2. 视觉（Vision）

**Commit:**
- **对比度**：所有文字 / 关键 UI 元素与背景对比 **≥ 4.5:1**（WCAG-AA）。art-bible 硬规则：UI 层明度必须高于世界环境段（D1–D5），B6 深蓝黑底 + B1 白字保证不沉入背景。
- **色盲双信道（P10，BLOCKING）**：任何语义色编码必须叠加非颜色冗余信道（形状/文字/图标/动画/边框）。已知冲突对处置见 art-bible §4.3 / interaction-patterns P10。关键实例：**LOSE 预演 = 红主题 + `lose_indicator`「会死」文字**（#10 CF-3）；**死亡预警 = 红背景 + 0.5Hz 边框闪烁**。
- **最小字号**：汉字 ≥12pt（低端触屏可读底线，#6/#10 锁定）；数字等宽像素字视觉 ≥16px（2× 整数倍渲染）；提示行小字 ≥10pt。所有文字 ≥1px 暗描边保证任意背景可读（art-bible §3.4）。
- **数字第一公民**：核心循环数字（HP/ATK/DEF）对比度永远高于图标（P1/art-bible §3.3）。

**Aspire:** 玩家可调字号档位；高对比模式（≥7:1）。

---

## 3. 运动 / 触控（Motor）

**Commit:**
- **触控热区 ≥ 44×44px(逻辑)**（art-bible §3.3）：视觉像素可更小，热区加透明缓冲。密集网格例外：16 列网格单格实际约 22dp，须实机校验误触率（#6 AC-VIS-5：≤10%）。
- **无时间压迫手势**：无需长按计时、无双击时间窗、无滑动速度门槛。确认用「再 tap」（P2）而非计时悬停（原悬停 200ms 方案已被 UX 废止）。
- **输入宽容**：tap 被忽略必有反馈（P7）；多指仅取首指防误触（P12）；结算/动画期锁输入防误触积压（P8）；× 纯取消零副作用（P5）。
- **单指可完成全部核心操作**：移动/攻击/取消/上楼/拾取均单指单点，无需多指协同。

**Aspire:** 触控热区放大选项（为运动障碍玩家）。

---

## 4. 认知（Cognitive）

**Commit:**
- **三秒上手（Pillar 3）**：核心交互零学习成本。隐藏 affordance（如「再 tap 确认」）必须用呈现层补可发现性（P6 高亮 + hint_label），不靠玩家猜。
- **渐进披露**：HUD 初始仅核心 4 槽，进阶资源解锁后弹入（P11）；DEF 首次战斗微教学出现一次（art-bible §7.6）。降低首屏认知负荷。
- **无阅读限速**：预演覆盖层、信息弹窗不自动消失（无强制阅读计时）；玩家自行决定何时确认/取消。
- **确定性可预测**：零随机数战斗（ADR-0006）+ 预演「说到做到」（#10）——结果可预测，降低焦虑型认知负荷。

**Aspire:** 可选的更详细教学层。

---

## 5. 听觉（Auditory）

**Commit:**
- **无唯音频承载的关键信息**：所有关键状态（受伤、胜负、成长）均有视觉信道（飘字/颜色/动画，P9）。音效是增强非必需——游戏在静音下完全可玩。

**Aspire:** 全面字幕系统（本游戏文本极少，优先级低）。

---

## 6. 动效（Motion）

**Commit:**
- **关键反馈不依赖快速闪烁**：胜负/危险用颜色+形状+文字多信道（P10），不单靠闪烁传达。
- **闪烁频率 ≤ 3Hz**（防光敏性癫痫，WCAG 2.3.1）：死亡预警 0.5Hz 边框闪烁远低于阈值；升级全屏白帧仅 1–2 帧瞬现（非重复闪烁）。
- **动画短促**：峰值 ≤128ms、消退 ≤200ms（art-bible §7.8），不构成持续运动负担。

**Aspire:** 全局 reduced-motion 开关（关闭飘字 45° 飘移、升级白帧、脉冲高亮等非必需动效）。MVP 不阻断，VS 评估。

---

## 7. 平台约束（Platform）

- **抖音小游戏 WebView**：无标准操作系统级屏幕朗读 / 辅助技术接口；Godot AccessKit（4.5+）在该环境支持未验证 → 屏幕朗读列 Aspire，不作 MVP 承诺。
- **纯触控单指**：无手柄/键盘焦点链；交互命中走 `_input()` Rect2（P4），不依赖 Godot GUI 焦点导航。故「键盘/手柄导航顺序」类 WCAG 项**不适用**（N/A，平台无此输入）。
- **低端设备**：可达性方案不得引入显著性能开销（如对象池覆盖节点承载高亮，ADR-0009）。

## 8. 各系统验收钩子

UI/UX 类 story 的 `/story-done` 须核对以下钩子（已分散在各 GDD/模式，此处汇总）：

| 钩子 | 模式/来源 | 验证点 |
|------|-----------|--------|
| 色盲双信道 | P10 / #10 CF-3 / art-bible §4.3 | LOSE 有非颜色标识；语义色对有形状/文字区分 |
| tap 拒绝反馈必装 | P7 / #6 AC-VIS-2 | 不可达 tap 有可见反馈，可注入接口可测 |
| 触控热区 ≥44px | P5/P12 / art-bible §7.8 | × 按钮、可点元素热区达标；网格实机误触率 ≤10%（#6 AC-VIS-5） |
| 对比度 ≥4.5:1 | art-bible §4.4 颜色主题表 | WIN/LOSE 主题、HUD 色位达标 |
| 字号 ≥12pt | #6/#10 / art-bible §7.8 | 汉字 ≥12pt、数字视觉 ≥16px |
| 可发现性双线索 | P6 / #10 CF-7 | 隐藏手势有高亮 + hint_label 两线索 |
| 闪烁 ≤3Hz | 本文档 §6 | 无 >3Hz 闪烁；死亡预警 0.5Hz |

## 9. 验证（Verification）

- **CI/自动化**：色盲双信道、字号、热区尺寸部分可由 UI 测试 + grep 断言（节点存在性、尺寸常量）。
- **手动/实机**：对比度、灰度截图可辨性、实机误触率（#6 AC-VIS-5：360dp 低端机 10 次 tap ≥9 命中）→ `production/qa/evidence/`。
- **灰度截图测试**：LOSE vs WIN 预演在灰度下仍明显可辨（#10 AC-CF-8b）。

## 10. Open Questions

1. **AccessKit 屏幕朗读在抖音 WebView 的可行性** `[VS 阶段 spike]`：Godot 4.5+ AccessKit 在抖音小游戏 WASM 环境是否可用未验证；当前列 Aspire。若可用且成本低，可在 VS 评估纳入。
2. **reduced-motion 全局开关范围** `[VS 评估]`：哪些动效纳入可关闭范围（飘字飘移/升级白帧/脉冲高亮），以及关闭后是否影响可发现性（P6 高亮若关闭，需替代静态线索）。
3. **art-bible §7.5 CTA 冲突**（同 interaction-patterns Open Q#3）：art-bible 仍画「进攻」CTA，与 #10 无 CTA 决策冲突，待 art-director 同步——影响本文档「可发现性」钩子的描述一致性。
4. **触控尺寸口径** 44px(art-bible) vs 40dp(GDD) vs 22dp(密集网格实际)——已在 §3 标注，最终以实机 #6 AC-VIS-5 基线为准。
